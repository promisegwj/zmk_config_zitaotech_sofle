# 自动层切换机制分析文档

## 问题背景

客户反馈：**指针自动向上飘，关闭右手后问题消失。观察发现是层一直停留在鼠标层，不会自动切回默认层导致的。**

---

## 核心机制：Input Processor + Input Listener

### 1. 设备树配置结构

#### 1.1 共享定义 (`zitaotech_sofle.dtsi:81-85`)
```dts
trackpoint_listener: trackpoint_listener {
    compatible = "zmk,input-listener";
    status = "disabled";  // 默认禁用
    device = <&trackpoint_split>;
};
```

#### 1.2 左手激活配置 (`zitaotech_sofle_left.dts:140-142`)
```dts
#define MOUSE_LAYER_ID 2

&trackpoint_listener {
    status = "okay";  // 在左手启用
    input-processors = <&zip_temp_layer MOUSE_LAYER_ID 400>;
};
```

**关键参数解读：**
- `MOUSE_LAYER_ID 2` - 目标层：鼠标层（层2）
- `400` - 400ms超时，停止移动后自动退回默认层

---

### 2. Input Processor: `zip_temp_layer`

**定义位置：** `zitaotech_sofle_left.dts:14-18`
```dts
zip_temp_layer: zip_temp_layer {
    compatible = "zmk,input-processor-temp-layer";
    #input-processor-cells = <2>;
};
```

**工作原理（基于ZMK标准机制）：**
1. **检测输入** - 监听 `trackpoint_split` 设备的输入事件
2. **自动切换到层2** - 有输入时立即切换到鼠标层
3. **400ms超时** - 停止输入400ms后自动切回默认层（层0）

## 数据流分析

### 右手小红点 → 左手层切换

```
右手（中央设备）:
  trackpoint硬件 (I2C 0x15)
    ↓
  trackpoint_0x15.c驱动
    ↓
  input_report_rel() → INPUT_REL_X/Y
    ↓
  trackpoint_split (input-split, ID=0)
    ↓ 蓝牙传输
左手（外设设备）:
  trackpoint_split (接收端)
    ↓
  trackpoint_listener (input-listener)
    ↓
  zip_temp_layer (input-processor)
    ↓
  [有输入] → 切换到层2（鼠标层）
  [无输入400ms] → 切回层0（默认层）
```

## 问题根源推测

### 假设1：右手持续误触发导致层不退出

**可能原因：**
1. **GPIO14干扰** - 右手小红点的MOTION引脚（GPIO0 PIN 14）受到电磁干扰
2. **I2C噪声** - I2C总线读取到错误数据包，导致 `dx != 0 || dy != 0`
3. **持续输入** - `trackpoint_0x15.c:205` 每5ms轮询，如果有干扰会持续发送input事件

**结果：**
- `zip_temp_layer` 不断收到输入事件
- 400ms超时计时器被持续重置
- **层一直停留在鼠标层，无法自动退出**

### 假设2：左手轨迹球干扰

**可能原因：**
1. GPIO1 PIN 5受到干扰，产生误触发（已通过上次修复缓解）
2. 但这不会影响层切换，因为左手轨迹球的 `bbtrackball_listener` 是 `status = "disabled"`

## 验证方法

### 方法1：日志监控（推荐）

在 `trackpoint_0x15.c:151` 的 `trackpoint_read_packet` 成功后增加日志：
```c
if (dx != 0 || dy != 0) {
    LOG_INF("小红点移动: dx=%d, dy=%d", dx, dy);
}
```

**观察：**
- 如果用户没动小红点，但日志持续输出 → 确认是干扰
- 关闭右手后日志停止 → 确认是右手干扰

### 方法2：增加小红点防抖

在 `trackpoint_0x15.c` 中增加类似左手轨迹球的防抖机制：
```c
#define TRACKPOINT_DEBOUNCE_MS 10  // 10ms防抖
static uint32_t last_valid_input_time = 0;

// 在发送input前检查
if (dx != 0 || dy != 0) {
    uint32_t now = k_uptime_get_32();
    if (now - last_valid_input_time < TRACKPOINT_DEBOUNCE_MS) {
        LOG_DBG("防抖过滤: %dms内重复输入", now - last_valid_input_time);
        return;  // 忽略
    }
    last_valid_input_time = now;
}
```

## 待修复方案（问题复现后执行）

### 方案1：加强右手小红点数据有效性检查（推荐）

**修改位置：** `trackpoint_0x15.c`

**思路：**
1. 增加防抖时间（10-20ms）
2. 过滤掉过小的移动值（|dx| < 2 && |dy| < 2）
3. 记录连续无效数据包计数，超过阈值则停止轮询

**优点：**
- 从源头解决干扰
- 不影响正常使用
- 不会误触发层切换

### 方案2：调整层切换超时时间

**修改位置：** `zitaotech_sofle_left.dts:142`

**调整：**
```dts
input-processors = <&zip_temp_layer MOUSE_LAYER_ID 200>;  // 从400ms改为200ms
```

**优点：**
- 更快退出鼠标层
- 减少干扰影响

**缺点：**
- 正常使用时可能过快退出（需要用户快速操作）

### 方案3：条件性禁用右手小红点（应急方案）

**修改位置：** `right_trackpoint.overlay`

**调整：**
```dts
&trackpoint_listener {
    status = "disabled";  // 临时禁用
};
```

**适用场景：**
- 确认右手硬件故障
- 用户不使用小红点功能

## 技术要点总结

1. **ZMK Input Processor机制**
   - `compatible = "zmk,input-processor-temp-layer"` - 标准临时层切换
   - 参数1：目标层ID
   - 参数2：超时时间（ms）

2. **Input Listener + Input Split**
   - 右手生成输入 → 左手接收并处理
   - 通过蓝牙传输input事件
   - 左手统一管理层切换逻辑

3. **关键文件位置**
   - 层切换配置：`zitaotech_sofle_left.dts:140-142`
   - 右手驱动：`trackpoint_0x15.c`
   - Input Processor定义：`zitaotech_sofle_left.dts:14-18`
